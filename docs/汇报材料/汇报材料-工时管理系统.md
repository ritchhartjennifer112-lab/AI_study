# 工时管理系统 — 汇报材料

> 铜牌仓设备 ETO 工厂 · 工时管理看板系统

---

## 1. 项目目标

**解决老板"实时看到设备/人员工时"的需求。**

当前痛点：
- 工时数据散落在 Excel 和个人微信/口头沟通中，无统一视图
- 标准工时靠经验估算，无法量化偏差
- IN3 ERP 工单数据与现场报工数据割裂

两周内交付一套基于 Streamlit 的网页看板，让：
- **组长** 每天 30 秒完成报工
- **老板** 打开网页就能看到设备/人员/效率全局
- **IE** 有数据支撑标准工时迭代修正

---

## 2. 系统架构（三层分离）

```
┌─────────────────────────────────────────┐
│              展示层 (Streamlit)            │
│  组长报工 · 今日总览 · 设备/人员/效率看板    │
└─────────────────────────────────────────┘
                    ↑
┌─────────────────────────────────────────┐
│           业务逻辑层 (Python Core)         │
│  工时切分 · 标准工时计算 · 偏差分析 · ERP导入 │
└─────────────────────────────────────────┘
                    ↑
┌─────────────────────────────────────────┐
│          数据层 (Excel/CSV 文件存储)        │
│  标准工时库 · 报工记录 · IN3工单导出         │
└─────────────────────────────────────────┘
```

**技术选型**：Python 3.10+ / Streamlit / Pandas / openpyxl / Plotly  
**数据存储**：Excel + CSV（适配无数据库工厂环境）  
**部署方式**：内网 `streamlit run app.py`，浏览器访问

## 3. 核心功能

### 3.1 组长每日报工（30秒流程）

1. 选择日期、员工
2. 系统自动从 IN3 拉取在制工单列表
3. 一键"平均分配"或"主任务优先"，可手动微调
4. 提交 → 自动保存到当日报工 CSV

支持**详细模式**：按时间段（上午/下午/加班）精确切分工时。

### 3.2 标准工时参数化计算（族A 已实现）

将设备装配分解为 8 个装配单元（A-01 ~ A-08），每单元有基准时间 × 驱动参数：

| 编码 | 装配单元 | 基准 | 驱动参数 |
|------|---------|------|---------|
| A-01 | 落地调平 | 4.5h | 模块数 |
| A-02 | 导轨安装 | 1.2h/m | 导轨长度 |
| A-03 | 丝杆装配 | 2.0h/套 | 丝杆套数 |
| A-04 | 支架固定 | 0.8h/个 | 支架数 |
| A-05 | 电机座固定 | 1.5h/个 | 电机座数 |
| A-06 | 夹钳装配 | 3.0h/模组 | 夹钳模组数 |
| A-07 | 传感器安装 | 0.5h/个 | 传感器数 |
| A-08 | 整机调试 | 8.0h | 元件数 |

支持四维修正系数：可达性、吊装方式、精度等级、设备尺寸。

### 3.3 实时看板（7 个维度）

| 看板页面 | 面向角色 | 核心指标 |
|---------|---------|---------|
| 今日总览 | 老板 | 在制工单数、今日总工时、人均工时、在岗人数 |
| 设备维度 | 老板/IE | 实际 vs 标准工时对比、偏差率、按产品族汇总 |
| 人员维度 | 组长/老板 | 每人总工时、出勤天数、日均工时、工时趋势 |
| 工单进度 | 老板 | 累计工时 vs 标准工时、进度百分比、累积趋势 |
| 效率看板 | IE/老板 | 全局效率、工单效率排行、每日工时趋势 |
| 标准工时健康度 | IE | 数据质量分布、基准时间在线编辑、修正建议 |

## 4. 数据质量分级

| 等级 | 标签 | 说明 |
|------|------|------|
| A | 实测 | IE 现场秒表测时 |
| B | 历史反推 | 从实际工时拆解得出 |
| C | IE 估算 | 基于 SOP 或经验估算 |
| D | 默认值 | 行业参考值 |
| E | 待验证 | 尚未确认 |

**当前状态**：族A 装配单元基准时间均为 C 级（IE估算），共 8 个单元。  
**升级路径**：积累 3 个月报工数据后，偏差分析模块自动建议修正 → 升级到 B 级。

## 5. 使用说明

### 组长：如何报工（3 步）

1. 打开网页 → 左侧导航栏 → 「组长报工」
2. 选择员工 → 勾选今日涉及工单 → 点「主任务优先」→ 微调数字
3. 点「提交」→ 完成

**首次使用前**：需先在「组长报工」页面上传一次 IN3 工单导出 Excel。

### 老板：如何看板

1. 打开网页 → 左侧导航栏
2. 「今日总览」→ 看当天全局
3. 「设备维度」→ 看哪些设备工时超标/节约
4. 「工单进度」→ 看各工单完成百分比
5. 切换日期范围查看历史趋势

### IE：如何修正标准工时

1. 「标准工时健康度」→ 查看修正建议
2. 在「装配单元基准库」中直接编辑基准时间
3. 保存后各看板自动使用新数据

## 7. 风险与退路

| 风险 | 退路 |
|------|------|
| 现场无法安装 Python | 用 `streamlit run` 一键启动，内网浏览器访问 |
| 标准工时偏差 > 40% | 族A 拆分为"小型机"和"中大型机"两个子族 |
| 组长不愿用 | 先让老板自己看数据，习惯后推动组长使用 |
| 历史数据不足 | 退为方案A：只做"设备-历史平均工时"汇总表 |
