## 本周最有价值推进

### 1. 标准工时从"人拍脑袋"升级为"AI 数学推导"

**之前的问题**：传统做法是 IE 凭经验给每个工单估一个标准工时——56 个工单根本估不过来，且无法解释"为什么是这个数"。

**AI 做了什么**：
- 读了 ERP 工单数据 + 历史报工数据 + BOM 物料清单，自动识别出 13 个产品子族
- 用统计学方法（子族 P25 下四分位数）为每个子族算保守基准
- 从产品描述中提取关键词，自动匹配难度系数（总装×1.3 / 改制×0.35 / 电气×0.65 …）
- 识别同一产品的历史生产次数，套 Wright 学习曲线给学习宽放
- 每个工单标注 A/B/C/D 置信度，低置信度数据明确告知"这个数还不太准"

**公式**：`标准工时 = 子族Q1 × 关键词系数 × 学习宽放`

**对比传统方式**：

| | 传统 IE | AI native |
|---|---|---|
| 覆盖 | 拍几个重点工单 | 56 个工单全量覆盖 |
| 可解释 | "我估的" | 每个数可追溯到子族/CV/系数 |
| 置信度 | 无 | A(16个)/B(21个)/C(16个)/D(3个) |
| 迭代 | 重估靠人记 | 新数据进来自动重算 |

## 工作单元故事

### 背景

冯家琛厂长给的任务很朴素："我要知道每台设备用了多少工时，每个人干了多少。"

但这个工厂的现状是：ETO 按单设计生产、无 MES 系统、工人满手油不填表。之前推过多次系统，都是半成品被 abandon。根因不是技术，是路径：在原始管理水平的工厂，软件是最后一个环节——先得有数据、有流程、有信任，系统才能落地。

### AI native 的做法

这次不同。不是"AI 帮我写代码"，而是 AI 作为整个工作流的分析引擎：

1. **AI 读数据**：直接读 CSV 报工文件 + ERP 导出 Excel + BOM 表，自动理解字段含义、发现数据问题
2. **AI 建模型**：没有标准工时？从历史数据反推——聚类产品族 → 计算统计基准 → 识别关键词 → 套学习曲线
3. **AI 出报告**：不是写死模板，而是根据数据量、数据质量、受众角色动态生成 HTML
4. **AI 做方案**：厂长 40 分钟录音 → AI 提取需求 → 归类 → 写方案设计文档
5. **人做判断**：公式选 Q1 还是中位数？厂长确认。子族分类对不对？IE 审核。优先级怎么排？人拍板。

**关键区别**：传统方式是人设计系统、AI 辅助写代码。AI native 是 AI 做分析、建模、生成，人做领域判断和质量把关。

### 本周关键迭代

| 轮次 | AI 做了什么 | 人的判断 |
|------|-----------|---------|
| 第一轮 | 按"本周数据"生成报告 | → 不对，要全量历史数据 |
| 第二轮 | 加搜索+排序+Tab | → 方向对，但标准工时是猜的 |
| 第三轮 | 分析 BOM+ERP+历史工时，提出子族Q1方案 | → 需要确认公式合理性 |
| 第四轮 | 解释 Q1 vs 中位数、学习曲线、关键词系数 | → 厂长确认用 Q1、加学习宽放 |
| 第五轮 | 厂长录音 → 提取需求 → 更新方案设计 | → 方向确认，下周继续 |

每一轮都是 AI 出分析 → 人到现场验证/沟通 → 反馈 → AI 调整。这个协作节奏比"写好需求→开发→验收"快得多。

## 附：需要什么支持

基于本周工作复盘，以下是 AI（Claude）继续有效推进这个项目需要的条件：

### 数据侧
- **每日报工持续稳定**：目前 5/21 仍缺。AI 能做异常检测，但不能替人报工。需要建立班组长每日强制报工流程
- **ERP 交期字段接入**：当前工单导出有此字段但未接入分析流。接入后 AI 可以直接做交货风险预判
- **ERP 物料/库存数据**：缺料分析目前靠手动记录，接入后可自动标注

### 沟通侧
- **厂长/现场人员定期反馈**：本周 5/22 晨会的 40 分钟反馈非常高效。AI 需要"真实声音"来校准方向，不能闭门造车
- **领域知识确认**：产品族分类、技能定义、加班阈值等需要人确认，AI 可以提案但不能替人拍板

### 方法论侧
- **关键决策点及时归档**：对话记录是 AI 的"记忆"，但重要的架构决策需要单独提炼。建议每做完一轮重大方向调整，花 5 分钟让 AI 写一份 200 字的决策备忘
- **失败路径保留**：被替换的方案代码不要删，改名归档（如 `_weekly_report_calc_v1_manual_std.py`），这样后来的 AI 能看懂"为什么这条路没走"

### 技术侧
- **Python 环境稳定**：当前环境可用，但 `python3` 命令不工作（需用完整路径），建议统一
- **部署方式选择**：当前纯 HTML 文件零部署，够用。后续如果要多人协作或实时数据，需要重新评估
