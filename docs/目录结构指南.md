> 任何新增文件、参考资料、数据、报告，先对照此表决定放哪。
> 最后更新：2026-06-03

---

## 项目定位

本项目有两层工作：

1. **开发层**：构建工厂工时管理 Streamlit 应用（代码）
2. **助理层**：作为生产助理处理日常数据，这些数据反哺开发

两层共用 `data/` 作为数据中枢。助理日常输入在 `data/助理日常/`，经处理后流入系统标准数据或报告输出。

## 各目录详细说明

### `core/` — 核心算法模块

- 存放：工时切分、标准工时计算、偏差分析、ERP导入等纯 Python 逻辑
- 不存放：任何业务数据、Excel、报表

### `pages/` — Streamlit 页面

- 存放：`1_组长报工.py`、`3_设备维度.py` 等前端页面
- 命名规则：`数字_中文名.py`，数字决定侧边栏顺序

### `tests/` — 单元测试

- 存放：`test_*.py`，对应 `core/` 里的模块

### `data/` — 数据中枢

```text
data/
├── 助理日常/                  ★ 助理日常数据（原始输入）
│   ├── meeting/               会议记录、录音转写、工单讨论
│   │   ├── 6.1兵哥六月份生产动员会/
│   │   └── 工单结构调整.txt
│   ├── plans/                 计划表、工单计划、出差工时
│   │   ├── 主计划表0602.xlsx
│   │   ├── 工单计划表.xlsx
│   │   └── 外勤人员出差工时.xlsx
│   └── reports/               考核方案等日常文档
│       └── 质量部六月考核指标方案.html
│
├── daily_reports/             组长每日报工 CSV（系统自动生成，保留英文名与SQL表名一致）
├── ERP导出/                   从 IN3 ERP 导出的工单 Excel
├── 原始档案/                   历史原始数据（只读档案）
│   ├── 钳工工时/
│   ├── 电工工时/
│   └── *.jpg / *.png          手写本拍照等原始凭证
├── 参考数据/                   参考数据
│   ├── 2026生产计划报表v1.xlsx    标准提前期（含排产规则）
│   ├── standard_time_baseline.xlsx
│   ├── 缺料需求清单-0520.xlsx
│   └── 电工工时/
├── 分析导出/                   分析脚本导出的 CSV
├── 归档/                       归档的非核心文件
├── factory.db                 主数据库
├── standard_time_library.xlsx 标准工时库（核心资产）
└── _plan.json                 计划配置
```

**规则**：
- `助理日常/`：助理日常收集的数据统一入口，按 meeting/plans/reports 分类
- `daily_reports/`：系统运行后自动生成，不手动往里丢文件。保留英文名，与SQLite表名一致
- `ERP导出/`：每次从 ERP 导出后覆盖旧文件，或按日期命名
- `原始档案/`：**只进不出**，历史档案不修改
- `参考数据/`：工厂提供的参考数据表，可被脚本读取

## 文件流转规则

| 场景 | 来源 | 放到哪 | 备注 |
|------|------|--------|------|
| 助理开会收集工单信息 | 现场/微信 | `data/助理日常/meeting/` | 录音转写、会议记录 |
| 助理拿到计划表 | 计划员 | `data/助理日常/plans/` | 工单计划、主计划 |
| 从 ERP 导出工单 | IN3 系统 | `data/ERP导出/` | 覆盖或日期命名 |
| 拿到手写本照片 | 现场拍照 | `data/原始档案/` | 原图，不压缩 |
| 工厂发来 PDF/Excel 资料 | 微信/邮件 | `docs/reference/` | 不改名，保留原文件名 |
| 助理数据处理成标准格式 | 脚本处理 | `data/参考数据/` | 如标准提前期表 |
| 跑完分析/报告 | 脚本生成 | `output/` | HTML/XLSX/CSV |
| 新的 IE 估算表 | 人工整理 | `data/standard_time_library.xlsx` | 直接覆盖或版本备份 |
| 汇报材料/周报 | 手动编写 | `docs/汇报材料/` | |
| Agent 对话记录 | Claude 导出 | `docs/conversations/` | |
| 临时调试输出 | 随手生成 | **删除** | 不提交到 git |

---

## 禁止事项

- 根目录不放 `.xlsx`、`.html`、`.jpg`、`.txt`（除了 `app.py` 和 `requirements.txt`）
- 不往 `data/原始档案/` 里放修改后的文件 — 这是档案区
- 不手动往 `data/daily_reports/` 里写 CSV — 系统自己写
- 临时文件（`out*.txt`、`error*.txt`）用完即删

---

## Git 提交范围

**应该提交的**：
- `core/`、`pages/`、`tests/`、`scripts/core/` 的代码改动
- `docs/` 里的指南、计划、汇报材料、设计文档
- `data/standard_time_library.xlsx`（核心资产）

**不应该提交的**：
- `data/daily_reports/*.csv`（运行期数据，已写进 `.gitignore`）
- `data/ERP导出/*.xlsx`（外部导入，随时变）
- `data/原始档案/`（档案太大且不常变）
- `data/助理日常/`（助理日常数据，不进 git）
- `docs/reference/`（工厂资料，不进 git）
- `output/`（可重新生成）
- `__pycache__`、`.pytest_cache`
