## 已讨论过的工时方法体系

### 标准工时公式

```
标准工时(ST) = 正常时间(NT) × (1 + 宽放率)
正常时间(NT) = 观测时间(OT) × 评比系数
工时效率 = 标准工时(ST) / 实际工时 × 100%
```

### ETO装配车间工时统计三大方案

1. **条码/PDA刷卡报工制**（主流MES逻辑，但需硬件+系统投入）
2. **数字化看板 + 磁吸/电子工牌**（过渡方案，适合长周期单任务）
3. **班组长日志"按比例切分"法**（最轻量，低代码表单）

### ETO参数化标准工时思路

不测整个项目，而是拆解为可复用的装配单元：

```
铜牌加工工时 = 铜牌数量 × (单牌基准 + 孔数×单位孔工时)
柜体装配工时 = 基础工时 + 门数×单门工时 + 隔板数×单隔板工时
元件安装工时 = Σ(各类型元件数量 × 单件安装基准)
二次接线工时 = 端子总数 × 单位端子接线时间
一次母线连接工时 = 母排段数 × 单段安装基准
调试/测试工时 = 按设备额定电流等级
```

### 关键纠正

- **工时效率公式**：分子是标准工时，分母是实际工时，不是反过来的
- **ETO最大浪费**：图纸/工艺变更导致的返工 > 缺料等待（跟大批量产线不同）
- **难度系数**：可用于成本核算，但不能用于工人绩效评估（公平性问题）
- **引入标准工时的正确顺序**：先搞准实际工时采集 → 积累数据反推标准工时 → 再做对比管控。不要跳步。

## 已明确的问题

1. 两周内不可能同时做"设备工时"和"人员工时"——数据采集无法落地
2. 老板真正能先要到的是一版可验证的**参数化标准工时基准**
3. 现有手工数据"水分大"的根因是班组长下班前凭记忆凑数，而非实时记录状态

## 已确定的设计决策

### 方案选择
- **选定方案B**：参数化标准工时计算器（Excel）
- **不选方案A**：历史数据有结构信息，可以做参数化
- **不选方案C**：两周做不完完整的MOD分析

### 产品族划分
- **族A（单机加工设备）**：冲剪机、折弯机、单头铣、双头铣、铜排加工设备 → 共享"机床式"结构
- **族B（龙门/仓储设备）**：铜排仓、尾料仓 → 共享"门架式"结构
- **族C（生产线设备）**：20工位抽屉线、打标机产线 → 共享"分布式产线"结构

### 数据资产确认
- 历史工时数据覆盖7种设备，**钳工数据有装配步骤记录**（足够做参数化）
- 电工数据（电柜+外围线）**没有装配步骤记录**，但用户决定也做参数化
- SOP只有冲剪机1份，覆盖族A约80%工步；族B/C无SOP，接受基于历史数据+IE估算

### 关键新增约束
- **大件搬运频繁且很重**：需在公式中加入重量驱动的吊装系数和疲劳宽放
- 用户已实际参与搬运，确认行吊使用和大件处理是重要时间驱动因素
